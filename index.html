API Reference version v2

https://api.pubpub.org

Introduction

This API provides full access to the PubPub database. All requests and responses are in the form of application/json. Requests should be made to the following URL paths added to the hostname https://api.pubpub.org. For example https://api.pubpub.org/user/travis.

Authentication

Access to secured URLs is granted by included an apiToken in the body of POST, PUT, and DELETE requests or an apiToken in the query of GET requests. The apiToken can be found on the settings page of each users profile (https://wwww.pubpub.org/user/travis/settings).

Contributing

This API documentation and the associated server code is all open source: https://github.com/pubpub/pubpub-api. Contributions, fixes, bug reports are welcome!

SignUp

Used to trigger a signup request for a given email.

/signup

get

Gets the associated signup data for a given hash. Most importantly, returns the email that was signed up with.

  • Request
  • Response

Query Parameters

  • hash: (string)

HTTP status code 200

Body

Type: application/json

Example:

{
  hash: "hfwk18jJGTE81maj12K0kal9BY",
  email: "email@example.com"
}

HTTP status code 500

Body

Type: application/json

Example:

Hash not valid
post

Creates a signup hash for a given email address. If the email has already been used (and account creation has not been completed), it will increment the count. On creation or increment, a signup email is sent.

  • Request
  • Response

Body

Type: application/json

Form Parameters
  • email: required (string)

    email of the account to be created

HTTP status code 200

Body

Type: application/json

Example:

true

HTTP status code 500

Body

Type: application/json

Example:

Email already tied to an account

  • Response

  • hash: required (string)

    first name of the new User to be created

    email that will be publicly displayed for the new User to be created

    github for the new User to be created

    orcid for the new User to be created

  • twitter: (string)

    twitter for the new User to be created

  • website: (string)

    website for the new User to be created

    googleScholar for the new User to be created
